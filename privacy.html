Privacy Policy

Last updated March 2026

The short version

All PDF processing happens locally on your Mac — nothing is uploaded anywhere.
We collect zero data — no analytics, no crash reports, no identifiers.
All PDF processing is offline. The only network requests are optional update checks via Sparkle and a one-time tool download.

No data collection

PDFloki does not collect, record, or transmit any personal information or usage data. This includes:

The PDF files you open or process
File names, metadata, or content
Crash reports or diagnostics
Usage patterns or feature analytics
Device identifiers or IP addresses

All processing is local

Every operation — compression, merging, splitting, watermarking, metadata editing — runs entirely on your Mac using native macOS frameworks. Your files never leave your device.

Minimal network access

PDFloki makes no network requests for PDF processing — all operations run locally. The app connects to the internet only for: (1) checking for app updates via Sparkle, which fetches an appcast file from GitHub, and (2) a one-time download of the qpdf compression tool from the Homebrew CDN if not already installed. There is no backend server, no cloud sync, and no telemetry.
